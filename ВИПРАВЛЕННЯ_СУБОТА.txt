📅 ПРОБЛЕМА:

При створенні замін на суботу НЕ з'являлась інформація про те, 
за яким днем тижня проводяться пари (наприклад: "пари за понеділок").

════════════════════════════════════════════════════════════

🔍 ПРИЧИНА:

У файлі replacement_form.py метод is_saturday() намагався 
визначити день тижня з тексту дати, але формат дати не містив 
інформацію про день тижня.

Формат дати: "18 листопада 2025 року" (без дня тижня)
День тижня зберігався окремо у змінній weekday: "субота"

Через це метод завжди повертав False, і інформація про суботні 
пари ніколи не відображалася.

════════════════════════════════════════════════════════════

✅ ВИПРАВЛЕННЯ:

Змінено метод is_saturday() у файлі replacement_form.py:

БУЛО (рядки 489-499):
```python
def is_saturday(self, date_str):
    try:
        date_parts = date_str.split()
        if len(date_parts) >= 3:
            day_name = date_parts[2].rstrip(',')
            return day_name.lower() == "субота"
        return False
    except:
        return False
```

СТАЛО:
```python
def is_saturday(self, date_str=None):
    try:
        # Перевіряємо день тижня з self.weekday (окрема змінна)
        if hasattr(self, 'weekday') and self.weekday:
            return self.weekday.lower() == "субота"
        return False
    except:
        return False
```
