📦 ВАРІАНТ 1: ВИКОРИСТАННЯ БЕЗ ІНСТАЛЯТОРА (НАЙПРОСТІШЕ)
────────────────────────────────────────────────────────────

Для розповсюдження програми БЕЗ інсталятора:

1️⃣ Створіть папку для розповсюдження:
   📂 Crisco_Portable/
   ├── 📄 Crisco_Optimized.exe
   ├── 📊 dataBase.mdb
   └── 📄 README.txt

2️⃣ Скопіюйте файли:
   • Скопіюйте dist\Crisco_Optimized.exe
   • Скопіюйте dataBase.mdb (ОБОВ'ЯЗКОВО!)
   • Додайте README.txt з інструкціями

3️⃣ Запакуйте в ZIP:
   • Стисніть папку в архів
   • Назвіть: Crisco_v1.0_Portable.zip

4️⃣ Розповсюдження:
   • Користувачі просто розпаковують ZIP
   • Запускають Crisco_Optimized.exe
   • Все працює без встановлення!

⚠️ ВАЖЛИВО:
• Файл dataBase.mdb завжди має бути в тій же папці, що й EXE
• На цільовому ПК має бути встановлений Microsoft Access Database Engine
• Інструкція для користувачів нижче

📦 ВАРІАНТ 2: СТВОРЕННЯ ПРОФЕСІЙНОГО ІНСТАЛЯТОРА
────────────────────────────────────────────────────────────

Для створення справжнього інсталятора потрібен Inno Setup:

КРОК 1: Встановіть Inno Setup
──────────────────────────────

1. Завантажте Inno Setup з офіційного сайту:
   🌐 https://jrsoftware.org/isdl.php

2. Скачайте версію: "Inno Setup 6.x.x" (стабільна версія)

3. Запустіть інсталятор і встановіть за замовчуванням
   (C:\Program Files (x86)\Inno Setup 6\)


КРОК 2: Створіть інсталятор
────────────────────────────

СПОСІБ А - Через графічний інтерфейс:

1. Відкрийте Inno Setup Compiler

2. Виберіть: File → Open → installer_setup.iss
   (файл знаходиться в папці вашого проєкту)

3. Натисніть: Build → Compile (або F9)

4. Інсталятор буде створено в папці:
   installer_output\Crisco_Setup.exe


СПОСІБ Б - Через командний рядок:

1. Відкрийте PowerShell в папці проєкту

2. Виконайте команду:
   & "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer_setup.iss

3. Інсталятор буде створено в папці:
   installer_output\Crisco_Setup.exe


СПОСІБ В - Через автоматичний скрипт:

1. Клацніть правою кнопкою на файл: build_installer.ps1

2. Виберіть "Виконати з PowerShell"

   Якщо PowerShell блокує скрипт, виконайте:
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   
   Потім знову запустіть build_installer.ps1

⚠️ ЯКЩО ПРОГРАМА НЕ ЗАПУСКАЄТЬСЯ

2️⃣ Виберіть версію:

4️⃣ Перезапустіть Crisco
